## ADDED Requirements

### Requirement: Record payment against a debt
The system SHALL allow users to record a payment against any individual debt entry. A payment requires: amount (required, must be > 0 and ≤ outstanding balance), currency (defaults to debt currency), date (required, defaults to today), and notes (optional).

#### Scenario: User records a valid payment
- **WHEN** user submits the payment form with a valid amount and date
- **THEN** system reduces the debt's outstanding balance by the payment amount
- **AND** system creates a corresponding expense record in the current monthly expense tab

#### Scenario: User attempts to overpay
- **WHEN** user enters a payment amount greater than the current outstanding balance
- **THEN** system displays a validation error and does not save

#### Scenario: User records a zero-amount payment
- **WHEN** user enters 0 as the payment amount
- **THEN** system displays a validation error and does not save

### Requirement: Payment creates linked expense
The system SHALL automatically create an expense record whenever a debt payment is saved. The generated expense SHALL have: amount and currency from the payment, date from the payment date, category set to "Debt Payment", merchant set to the debt's source name, notes referencing the debt (e.g., "Payment toward [source]"), and a receipt URL left empty.

#### Scenario: Payment expense appears in expense list
- **WHEN** a debt payment is saved
- **THEN** the generated expense appears in the Expenses view with category "Debt Payment"
- **AND** it appears in the monthly expense tab in Google Sheets

#### Scenario: Payment expense appears in dashboard totals
- **WHEN** a debt payment falls within the dashboard's selected period
- **THEN** its amount is included in the total spending and category breakdown

### Requirement: Mark debt as fully paid
The system SHALL automatically mark a debt as "paid" when its outstanding balance reaches zero. Paid debts SHALL be visually distinguished (e.g., strikethrough or "Paid" badge) and may be hidden or shown via a toggle.

#### Scenario: Final payment clears a debt
- **WHEN** a payment reduces the outstanding balance to exactly zero
- **THEN** the debt entry is marked as paid
- **AND** the source card updates its aggregated balance (removing the paid debt's contribution)

#### Scenario: Toggle to show/hide paid debts
- **WHEN** user activates the "Show paid" toggle in the Debts view
- **THEN** paid debt entries become visible in their source cards

### Requirement: View payment history for a debt
The system SHALL allow users to view a list of all payments recorded against an individual debt, showing date, amount, and notes for each.

#### Scenario: User views payment history
- **WHEN** user expands a debt entry's payment history
- **THEN** system displays all payments in reverse-chronological order with date, amount, and notes

#### Scenario: No payments recorded yet
- **WHEN** user views payment history for a debt with no payments
- **THEN** system displays an empty state message

### Requirement: Debt Payment category in expense system
The system SHALL include "Debt Payment" as a built-in expense category available in filters, dashboard charts, and the category breakdown.

#### Scenario: Filter expenses by Debt Payment category
- **WHEN** user selects "Debt Payment" in the category filter on the Expenses view
- **THEN** only expenses generated by debt payments are shown

#### Scenario: Debt Payment appears in category chart
- **WHEN** the dashboard period includes debt payment expenses
- **THEN** "Debt Payment" appears as a segment in the spending-by-category doughnut chart
